# Техническое задание

## Проект: UMKOVO — AI-помощник для обучения

UMKOVO — Telegram-бот, который помогает изучать темы через:

- структурированные конспекты;
- тестирование;
- анализ ошибок;
- обучение по слабым разделам.

Сервис не выполняет работы за пользователя, а помогает понять материал.

Основной цикл работы:

`Тема / файл → План → Конспект → Тест → Анализ → Дообучение`

---

## 1. Архитектура проекта

### Стек

- Python
- Aiogram 3
- SQLAlchemy async
- SQLite (позже PostgreSQL)
- RAG
- LLM API

```text

  pdf_service.py
  keygen_service.py
  access_service.py

infrastructure/
  db.py
  db_models.py
  repositories/
```

## 2. One Screen интерфейс

Бот использует одно основное сообщение. Все экраны обновляются через `edit_message_text`, чтобы не засорять чат.

Класс `OneScreen` отвечает за:

- `edit_message_text`;
- fallback на `send_message`;
- хранение `main_message_id`.

## 3. Доступ по ключу

Без ключа пользователь не может пользоваться ботом и не должен видеть меню.

### Стартовый экран

Приветствие зависит от времени суток:

- Доброе утро
- Добрый день
- Добрый вечер
- Доброй ночи

Если есть имя: `Доброе утро, Никита`.

Если имени нет: `Доброе утро, @username`.

Текст:

`🔑 Введите лицензионный ключ.`

Формат ключа: `UMK-####-####`

`Если у вас нет ключа, запросите его у администратора.`

Кнопки:

- `🔑 Ввести ключ`
- `📩 Запросить ключ`

### Запрос ключа

Экран содержит инструкцию.

Кнопки:

- `👤 @mcknmf`
- `👤 @Usikling`
- `⬅ Назад`

## 4. Главное меню

После активации ключа появляется меню:

- `▶ Продолжить`
- `📘 Создать конспект`
- `📚 Ваши работы`
- `👤 Профиль`

## 5. Профиль

Показывает:

- имя пользователя;
- статус доступа;
- срок действия ключа;
- статистику обучения.

Пример:

```text
👤 Никита

Доступ: активен
Ключ действует до: 12.05.2026

Изучено тем: 8
Средний результат тестов: 78%
```

## 6. Создание конспекта

Пользователь может:

- ввести тему;
- загрузить файл.

Поддерживаемые форматы: `txt`, `pdf`, `docx`.

## 7. Формат конспекта (цели)

После темы пользователь выбирает формат объяснения:

- `⚡ Кратко`
- `📚 Подробно`
- `🧠 Простым языком`
- `📌 Шпаргалка`

Важно:

- не показывать проценты сжатия;
- не показывать технические параметры вида «до 500 слов».

Пользователь выбирает только цель.

## 8. Генерация конспекта

Pipeline генерации:

1. Нормализация темы.
2. Уточнение темы.
3. Генерация плана.
4. Проверка плана.
5. Поиск контекста RAG.
6. Генерация разделов.
7. Очистка текста.
8. Выравнивание разделов.
9. Сборка конспекта.
10. Генерация теста.

## 9. План конспекта

План должен быть предметным.

Плохой пример:

- Что это
- Как работает
- Применение

Хороший пример:

- Классы и объекты
- Наследование
- Инкапсуляция
- Полиморфизм

### Экран плана

Кнопки:

- `📘 Создать конспект`
- `🔁 Перестроить план`
- `✏ Изменить тему`
- `🏠 В меню`

## 10. Статус генерации

Во время генерации показывать прогресс, обновляя одно сообщение:

- Анализируем тему
- Строим план
- Подбираем материалы
- Генерируем конспект

## 11. Конспект

Конспект разбит на разделы. Каждый раздел отображается отдельно.

### Формат текста

Текст должен использовать:

- жирные заголовки;
- абзацы;
- списки;
- курсив;
- моноширинный текст;
- блоки кода.

Запрещено:

- сырые markdown fences;
- текст типа «Источник»;
- «мусор» из RAG.

### Равномерность разделов

Разделы должны быть примерно одинаковыми по объёму.

### Навигация разделов

Кнопки:

- `⬅ Предыдущий раздел`
- `➡ Следующий раздел`

Индикатор: `Раздел 2 из 6`.

## 12. Изменение раздела

Кнопка: `✏ Изменить`

После нажатия доступны:

- `✂ Кратче`
- `📚 Больше`
- `🧠 Переписать проще`

## 13. Тест

Тест связан с разделами конспекта.

Каждый вопрос хранит:

- `text`
- `options`
- `correct_answer`
- `explanation`
- `section_id`

### Показ теста

Вопросы показываются по одному.

### После теста

Показывается результат:

```text
Результат: 6 из 8

Правильных: 6
Ошибок: 2
```

## 14. Разбор ответов

Пользователь видит:

- вопрос;
- свой ответ;
- правильный ответ.

Пример:

```text
Вопрос: Что такое инкапсуляция?

Ваш ответ: скрытие данных
Правильный ответ: механизм ограничения доступа

❌ Неверно
```

Кнопки:

- `⬅`
- `➡`
- `📖 Повторить слабый раздел`
- `🏠 В меню`

## 15. Обучение по слабым разделам

Система анализирует ошибки, определяет самый слабый раздел и:

- берёт текст раздела;
- переписывает его простым языком;
- показывает объяснение;
- предлагает повтор.

## 16. Ваши работы

Название: `📚 Ваши работы`

Описание: `В данном разделе хранится история ваших работ.`

### Список работ

Работы отображаются только кнопками.

Формат кнопки:

```text
📄 ООП в Python
Конспект • кратко • 80%
```

Не показывать:

- `ready`
- `in_progress`
- `status`

### Пагинация

Горизонтальная:

- `⬅`
- `➡`

Счётчик страниц текстом: `Страница 1/3`

Кнопка: `🏠 В меню`

## 17. Открытие работы

При нажатии на работу пользователь сразу попадает в конспект.

## 18. Конспект из файла

Файл:

- извлекается;
- очищается;
- по нему строится план;
- генерируется конспект.

## 19. PDF экспорт

Кнопка: `📄 Скачать PDF`

PDF должен содержать:

- кириллицу;
- заголовки;
- структуру;
- списки;
- блоки кода.

## 20. Админ-панель

Команда: `/admin`

Функции:

- создание ключей;
- выдача ключей;
- просмотр ключей;
- изменение;
- удаление.

### Уведомления

Если ключ пользователя:

- выдан;
- изменён;
- продлён;
- отключён,

то пользователь получает уведомление.

Кнопка: `Закрыть`.

## 21. Callback-схема

Примеры:

- `menu:create`
- `menu:works`
- `menu:profile`
- `topic:format:brief`
- `topic:format:detailed`
- `plan:confirm`
- `plan:rebuild`
- `summary:next`
- `summary:prev`
- `test:start`
- `test:answer`
- `works:page:2`

## 22. Итоговый пользовательский flow

Без ключа:

`Start → ввод ключа`

После активации:

- Главное меню;
- создание темы: `Тема → формат → план → генерация → конспект`;
- после конспекта: `Тест → результат → обучение`;
- история: `Ваши работы → список → открыть`.

## 23. Требования к реализации

Нужно:

- сохранить архитектуру;
- не ломать существующие функции;
- использовать текущие сервисы;
- добавить недостающие методы;
- отдавать полный код изменённых файлов при передаче изменений.
